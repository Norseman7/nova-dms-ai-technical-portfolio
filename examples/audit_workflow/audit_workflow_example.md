# Audit Workflow Example

## Sanitized Audit Flow

```text
1. Define audit purpose.
2. Define target project/folder.
3. Define scope and non-scope.
4. List protected areas.
5. Read required authority documents.
6. Inspect files and evidence.
7. Classify findings by severity.
8. State root cause for each finding.
9. Record commands/static checks performed.
10. Create report with deferred human checks if needed.
11. Preserve report for future review.
12. Do not remediate without a separate approved task.
```

## Review Value

This workflow demonstrates deterministic governance. Audits are not vague reviews; they have boundaries, report formats, protected areas, and completion criteria.
