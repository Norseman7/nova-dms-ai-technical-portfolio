# Combined System View

## Purpose

This view shows how **NOVA**, **DMS**, and the **Figma workflow** fit together as a technical portfolio.

It presents the combined system as a practical operating model for AI-assisted development, governance, validation, and decision-support architecture.

---

## High-Level System View

```text
DMS governance layer
  -> task scope, handovers, audits, reports, documentation control

NOVA technical system
  -> acquisition, raw authority, DATABASE, ANALYSIS, Brain Store, AI reasoning

Figma workflow support
  -> UI/design review, naming consistency, scoped design-to-development handoff
```

---

## System Roles

| System | Role | Portfolio signal |
| --- | --- | --- |
| **NOVA** | Core modular AI-enabled decision-support architecture. | Complex data, event, validation, and AI-boundary design. |
| **DMS** | Development management and governance layer. | Continuity, auditability, architecture protection, and AI-assisted work control. |
| **Figma workflow** | Supporting UI/design quality-control workflow. | Controlled design review and Codex implementation alignment. |

---

## Control Flow

```text
Pål defines objective and acceptance
  -> DMS captures task, document, and audit context
  -> ChatGPT / Codex support specification or implementation planning
  -> NOVA architecture receives bounded changes or documentation
  -> validation and human review confirm the result
  -> handover / report preserves continuity
```

---

## Evidence Theme

The combined system shows more than technical build activity.

It shows an operating model for complex AI-assisted engineering:

- define scope
- preserve authority
- implement in bounded layers
- validate evidence
- document limitations
- keep human approval in the loop
