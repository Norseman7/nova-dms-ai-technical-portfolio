# Capability Matrix

| Capability | Evidence from portfolio | Why it matters |
| --- | --- | --- |
| Translate complex domain problems into AI/software architecture | NOVA models market behavior as raw data, derived datasets, event modules, context tags, outcomes, correlations, and future evidence accumulation. | Shows ability to turn ambiguous expert judgment into explicit system contracts. |
| Modular AI-enabled system design | NOVA separates acquisition, raw authority, DATABASE, derived indexes, ANALYSIS, Brain Store, interpretation, and UI. | Supports maintainability, testing, and safe evolution. |
| High-volume data architecture | The operating layer favors derived indexes, summaries, manifests, and immutable snapshots over repeated raw scans. | Demonstrates scalability thinking for time-series/order-flow data. |
| Event-driven analysis | Event families include trigger events, reference interactions, value-based signals, market-state classifiers, and order-flow memory zones. | Shows extensible analytical design beyond one detector. |
| Validation and traceability | Standard outcome envelopes, timestamp cutoff safety, source lineage, rule versions, and diagnostics are built into the architecture. | Enables evidence-based review and repeatable analysis. |
| Human-in-the-loop safeguards | Human review is required for visual validation, runtime behavior, evidence promotion, and final acceptance. | Prevents overreliance on automation or AI-generated conclusions. |
| AI-assisted development workflow | ChatGPT and Codex are used under bounded prompts, review gates, task discipline, and acceptance criteria. | Shows controlled use of AI as an accelerator rather than uncontrolled delegation. |
| Governance and auditability | DMS provides audit templates, document control, handover workflow, project context, and report discipline. | Supports continuity across long-running technical work. |
| Documentation discipline | The portfolio maps source evidence into clear review paths, capability tables, architecture views, and limitations. | Helps both human reviewers and AI screeners identify substance quickly. |
| Production-readiness mindset | Non-goals, fail-closed behavior, lifecycle states, immutable inputs, quality gates, and architecture protection are explicit. | Shows awareness of operational risk and maturity boundaries. |
| Stakeholder and operational decision support | NOVA is framed as decision support with structured evidence, uncertainty, context, and review, not autonomous execution. | Aligns technical architecture with operational responsibility. |
