# AI Technical Portfolio - Sanitized Evidence Repository

## Repository Purpose

This repository is a curated, sanitized evidence package supporting an AI Solutions Engineer application portfolio. It provides selected publication-safe evidence of controlled AI-assisted development, NOVA architecture, DMS governance, traceability, validation discipline, human-in-the-loop safeguards, and selected Figma/design workflow automation.

## What This Repository Is

- A GitHub-ready evidence folder for review.
- A companion to the technical portfolio PDF in `portfolio/`.
- A curated set of Markdown evidence files and one final portfolio PDF.
- A navigation aid for technical reviewers, hiring stakeholders, interviewers, and AI-assisted screening tools.

## What This Repository Is Not

- It is not the real NOVA working repository.
- It is not the real DMS working repository.
- It is not a runtime repository.
- It is not a full source-code publication.
- It is not a commercial trading bot or autonomous decision system.
- It does not publish raw market data, runtime logs, credentials, local caches, or protected implementation material.

## Relationship To The Technical Portfolio PDF

The final PDF in `portfolio/` is the primary narrative artifact. The surrounding folders provide supporting evidence in a reviewer-friendly structure: architecture notes, capability mapping, validation/traceability models, governance examples, and sanitized workflow examples.

## NOVA Summary

NOVA is presented as an AI-enabled decision-support and evidence architecture under active development. Its portfolio value is in the architecture: source authority, high-volume data handling, derived datasets, event modules, validation boundaries, Brain Store planning, AI reasoning safeguards, and human review.

NOVA is not presented as production deployed, commercially released, autonomous trading software, or a guarantee of detector performance.

## DMS Summary

DMS is presented as reusable development governance for controlled AI-assisted software work. It supports continuity through handovers, audit workflows, documentation control, architecture protection, and explicit human approval points.

## Controlled ChatGPT / Codex Workflow Summary

ChatGPT and Codex are positioned as scoped assistants. They support drafting, inspection, implementation planning, documentation, and selected bounded changes, while human ownership remains responsible for architecture direction, requirements, validation decisions, and final acceptance.

## Figma / Design Workflow Summary

The Figma plugin evidence is included as supporting tooling. It demonstrates design quality-control workflow, naming consistency, structured Figma-to-Codex handoff, and reduced ambiguity between design review and implementation.

## Validation And Traceability Summary

The evidence emphasizes traceability from AI-supported conclusions back through event/module outputs, derived datasets, indexed sessions, source evidence, validation references, and human review. Validation is treated as a control layer rather than a metric slogan.

## Maturity, Limitations, And Non-Claims

This repository intentionally uses conservative language. NOVA is under active development. DMS is reusable governance/support tooling, not an enterprise-certified platform. The Figma workflow is supporting tooling, not a separate main product equal to NOVA. The repository excludes sensitive/noisy material and should be manually reviewed before publication.
